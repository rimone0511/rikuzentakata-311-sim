# シミュレーション変更のレビュー

このプロジェクトは防災教育・伝承のための追体験作品であり、娯楽ゲームではない。
レビューは「倫理ルール」→「技術的な不変条件」→「動作確認」の順で行う。
指摘には必ず該当ファイル・行の根拠を添える。確信が持てない指摘は「要確認」と明記する。

## 1. 倫理ルール(1つでも違反したら実装を止めてユーザーに報告)

- [ ] ゲームオーバー・失敗演出がない(被災時は白転+事実の表示のみ。効果音・赤い画面・
      スローモーション等の演出を追加していない)
- [ ] スコア・ランキング・タイム表示・称賛メッセージがない(NPC統計は内部値のみ)
- [ ] 実在個人を特定できる表現がない(NPCは匿名の合成キャラ5類型のみ。名前・顔・
      実在の住所と結びつく設定を足していない)
- [ ] 音はすべて Web Audio 合成で、ホラー的な演出音・BGM を足していない
- [ ] 新しいデータソースを使う場合、合法・公開・規約準拠であること
      (Googleストリートビューのスクレイピングは規約違反のため禁止。過去に合意済み)
- [ ] 史実に関する文言(犠牲者数・時刻・場所)を変えた場合、出典が公開資料であること

## 2. 技術的な不変条件(ARCHITECTURE.md と照合)

- [ ] 座標変換は terrain.latLonToWorld / heightAt を使っている(自前の変換式を書いていない)
- [ ] simTime の持ち主は main.js のまま(他モジュールが独自に時刻を進めていない)
- [ ] 倍速(timeScale)時も移動距離が実時間と一致する(player 0.05s / npc 0.1s の
      内部分割を壊していない)
- [ ] 白転条件(depth > 0.3 かつ waterLevel > 1.0)の後段を消していない
      (消すと平常の海・川で白転する)
- [ ] NPC・建物の生成に Math.random() を混ぜていない(mulberry32 の決定性を維持)
- [ ] 衝突判定は town.resolveCollision に一本化されたまま
- [ ] assets の fetch は cache: 'no-cache' のまま
- [ ] 追加コードが Node.js を前提にしていない(ブラウザ+Python 3.11 標準ライブラリのみ)
- [ ] tsunami.js の TIMELINE を変えた場合、EVENTS(HUD表示)も整合している

## 3. 動作確認(可能なら実施、できなければ「未検証」と明記)

開発サーバー(python -m http.server 8000)を起動し、ブラウザで確認する。
コンソールの `__sim` で時刻を操作できる。

- [ ] 起動してエラーなく地形・街・NPCが表示される(コンソールにエラーがない)
- [ ] `__sim.simTime = 2700` で第一波直前に飛び、浸水が市街地へ広がる
- [ ] 史実整合の基準ケース(PLAN.md 工程6の検証結果と同じ結果になるか):
      旧駅前→本丸公園 14:49出発 → 生存 / 市民会館前で待機 → 15:25頃にのまれる
- [ ] 被災時: 静かに白転し、事実のみ表示される(演出が増えていない)
- [ ] 16:46 まで生存: 静かな結びが表示される(称賛表現がない)

## 4. 報告形式

- 結論(問題なし / 要修正 n件 / 要確認 n件)を最初に1行
- 指摘は深刻度順に、根拠(ファイル:行)つきで
- 「未検証」の項目は理由とともに明記する(検証したふりをしない)
